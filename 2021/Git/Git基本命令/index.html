仓库级别的配置之后会存储在仓库中.git目录下的config文件中

用户级别的配置之后会存储在用户根目录下.gitconfig文件中

commit后边的一串字符串即为此次提交的hash值，此次提交的唯一标识，HEAD即为指针，表示当前所在的版本

版本前进或后退

版本前进后退的本质是移动指针，即修改HEAD所指向的commit提交

基于索引值操作[推荐]

前进后退都可以，只需要根据索引值即可

reset的三个参数
  • soft 仅仅在本地库移动指针，不会更改暂存区和工作区
  • hard 在本地库移动指针，重置暂存区和工作区
  • mixed 在本地库移动指针，并进行重置暂存区，不会更改工作区
使用^符号[了解]

只可以后退

使用~符号[了解]

是^的升级版，也是只能后退，但是由于如果后退版本多的话使用^比较麻烦，使用之后跟后退的步数比较简洁一点

注意：git reset是移动指针指向某个版本，此时的commit_id也会变为所要到达的版本的commit_id，而且所有的状态都变成了该版本的状态

git revert表示的是回滚某个提交，只针对于该提交进行操作，不影响该提交的前后版本，且会生成新的commit_id

文件差异比较

可以使用diff来比较工作区和暂存区的文件

也可以来比较不通过版本之间的差异
